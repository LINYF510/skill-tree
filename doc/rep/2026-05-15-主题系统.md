# 主题系统实现报告

## 实现概述

为 Skill-Tree 项目实现暗色/亮色双主题切换系统，从硬编码颜色常量迁移到基于 `LocalThemeColors` CompositionLocal 的动态配色方案。支持三种主题模式（暗色/亮色/跟随系统），通过 SharedPreferences 持久化用户选择。

## 关键代码片段

### Color.kt 架构

```kotlin
object DarkColors { val Background = Color(0xFF0D1117); ... }
object LightColors { val Background = Color(0xFFFFFFFF); ... }
enum class ThemeMode { DARK, LIGHT, SYSTEM }
data class ThemeColors(background, surface, primary, ability, resource, link, ai, textPrimary, textSecondary, error, warning)
fun getThemeColors(mode: ThemeMode, isSystemDark: Boolean): ThemeColors
```

### Theme.kt 架构

```kotlin
val LocalThemeColors = staticCompositionLocalOf { ThemeColors.fromDark() }
@Composable
fun SkilltreeTheme(themeMode: ThemeMode = ThemeMode.DARK, content: ...) {
    CompositionLocalProvider(LocalThemeColors provides themeColors) { MaterialTheme(...) }
}
```

### 迁移模式

```kotlin
// Composable 中
val colors = LocalThemeColors.current
Text(color = colors.textPrimary)

// 非 Composable 私有函数中
private fun SomeHelper(colors: ThemeColors, ...) {
    Text(color = colors.textPrimary)
}
```

### 主题切换 UI

SettingsScreen 新增「🎨 外观」分组，包含主题模式选项。点击弹出 AlertDialog + RadioButton 三选一（暗色/亮色/跟随系统），选择后立即持久化到 UserPreferences 并通过 onThemeChanged 回调刷新 Activity。

## 已知问题

1. `MarkdownText` 中 `TextView.setTextColor` 仍硬编码 `#E6EDF3`，需通过 ThemeColors 传入（需修改 AndroidView factory 参数）
2. 旧常量标记 `@Deprecated` 但仍保留在 Color.kt 中，后续版本可移除
3. `UserPreferences` 使用 `SharedPreferences.apply()` 异步写入，极端情况下可能丢失最后一次主题选择
4. 主题切换后整个 Activity 的 `setContent` 会重新执行，所有 Composable 重新组合
