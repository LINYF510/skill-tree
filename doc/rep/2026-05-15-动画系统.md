# 2026-05-15-动画系统

## 实现概述

为 Skill-Tree 项目实现了完整的动画系统，涵盖 7 种动画效果，全部使用 Compose 原生动画 API，未引入第三方动画库。

### 动画效果清单

| # | 动画效果 | 实现方式 | 时长 |
|---|---------|---------|------|
| 1 | 节点创建动画 | `Animatable` + `spring` (scale 0→1 + alpha 0→1) | ~400ms |
| 2 | 节点生长动画 | `Animatable` + `tween` (逐层展开，子节点从父节点位置滑出) | 800ms + depth×200ms |
| 3 | 节点选中脉冲 | `infiniteTransition` + `animateFloat` (边框发光脉冲) | 600ms 循环 |
| 4 | 连线绘制动画 | `PathMeasure.getSegment` (从父到子逐渐绘制) | 随生长动画 |
| 5 | 搜索高亮呼吸 | `infiniteTransition` + `animateFloat` (边框呼吸闪烁) | 600ms 循环 |
| 6 | 成就解锁动画 | `AnimatedVisibility` + `slideInVertically` + `scaleIn` + `spring` | 400ms |
| 7 | 页面转场 | NavHost `enterTransition`/`exitTransition` + `slideIntoContainer` | 400ms |

### 架构设计

- **AnimationConfig**: 集中管理所有动画时长常量，便于全局调整
- **AnimationParams**: Canvas 绘制时传递的动画状态值数据类，将多个动画参数打包传递
- **TreeNode.depth**: 新增深度字段，支持逐层动画延迟计算
- **buildTreeNodes**: 修复 `!!` 非空断言，改用 `?: return@forEach`

### 树生长动画（逐层展开）

```kotlin
LaunchedEffect(treeNodes.isNotEmpty()) {
    if (treeNodes.isNotEmpty() && !hasPlayedGrowthAnimation) {
        hasPlayedGrowthAnimation = true
        growthProgress.snapTo(0f)
        growthProgress.animateTo(
            (maxDepth + 1).toFloat(),
            animationSpec = tween(
                durationMillis = AnimationConfig.NODE_GROW_DURATION + maxDepth * AnimationConfig.NODE_GROW_LAYER_DELAY,
                easing = FastOutSlowInEasing
            )
        )
    }
}
```

### 节点创建动画（弹簧效果）

```kotlin
LaunchedEffect(uiState.nodes) {
    val currentNodeIds = uiState.nodes.map { it.id }.toSet()
    val newIds = currentNodeIds - previousNodeIds
    if (hasPlayedGrowthAnimation && previousNodeIds.isNotEmpty() && newIds.isNotEmpty()) {
        newlyCreatedNodeId = newIds.first()
        creationScale.snapTo(0f)
        creationScale.animateTo(
            1f,
            animationSpec = spring(
                dampingRatio = Spring.DampingRatioMediumBouncy,
                stiffness = Spring.StiffnessMedium
            )
        )
        delay(500)
        newlyCreatedNodeId = null
    }
    previousNodeIds = currentNodeIds
}
```

### 连线绘制动画（PathMeasure 逐步绘制）

```kotlin
if (connectionProgress >= 1f) {
    drawPath(path = path, color = LinkOrange, style = Stroke(width = 3f))
} else {
    pathMeasure.setPath(path, false)
    val partialPath = Path()
    pathMeasure.getSegment(0f, pathMeasure.length * connectionProgress, partialPath, true)
```

### 成就解锁动画（底部滑入+缩放弹出）

```kotlin
AnimatedVisibility(
    visible = true,
    enter = slideInVertically(
        initialOffsetY = { it / 2 },
        animationSpec = spring(
            dampingRatio = Spring.DampingRatioMediumBouncy,
            stiffness = Spring.StiffnessMedium
        )
    ) + fadeIn(animationSpec = tween(AnimationConfig.ACHIEVEMENT_ENTER_DURATION))
    + scaleIn(
        initialScale = 0.5f,
        animationSpec = spring(
            dampingRatio = Spring.DampingRatioMediumBouncy,
            stiffness = Spring.StiffnessMedium
        )
    ),
    exit = slideOutVertically(targetOffsetY = { it / 2 })
    + fadeOut(animationSpec = tween(AnimationConfig.ACHIEVEMENT_ENTER_DURATION))
)
```

## 已知问题

1. `Icons.Filled.ArrowBack` 已弃用，建议后续迁移到 `Icons.AutoMirrored.Filled.ArrowBack`（已有代码，非本次引入）
2. 节点创建动画仅支持单节点创建，同时创建多个节点时只有第一个会播放创建动画
3. `infiniteTransition` 动画（搜索脉冲、选中脉冲）始终运行，即使没有搜索或选中状态，对性能有轻微影响
4. 生长动画仅在首次加载时播放，切换 Tab 后返回不会重播
5. `PathMeasure` 在某些低端设备上可能有性能影响，连线数量较多时需关注
