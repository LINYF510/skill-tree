## 一、实现概览

为 Skill-Tree 应用实现完整的国际化（i18n）支持，包括中英文双语资源、语言切换管理器、全量硬编码文本迁移，以及错误系统的资源 ID 重构。

---

## 二、核心组件

### 2.1 字符串资源文件

| 文件 | 语言 | 说明 |
|------|------|------|
| `res/values/strings.xml` | 英文（默认） | ~162 行，覆盖全部 UI 文本 |
| `res/values-zh/strings.xml` | 中文 | 与英文一一对应 |

**资源分类**：
- 导航、技能树首页、节点详情、创建节点
- 标签、添加链接、添加附件、设置
- 统计、成就、通用、错误消息、引导页

**格式字符串示例**：
- `tree_export_success` → "Exported to %s" / "已导出到 %s"
- `tree_export_failed` → "Export failed: %s" / "导出失败: %s"
- `link_confirm` → "Confirm (%d)" / "确认 (%d)"
- `tag_create_hint` → "Create tag \"%s\"" / "创建标签 \"%s\""

### 2.2 LocaleManager

**路径**: `core/data/preferences/LocaleManager.kt`

```kotlin
@Singleton
class LocaleManager @Inject constructor(
    @ApplicationContext private val context: Context
) {
    var languageTag: String?          // SharedPreferences 持久化
    fun getCurrentLocale(): Locale    // 获取当前 Locale
    fun applyLocale(baseContext: Context): Context  // Context 包装

    companion object {
        const val LANG_ZH = "zh"
        const val LANG_EN = "en"
        val LANG_SYSTEM: String? = null  // 跟随系统
    }
}
```

- 使用 `SharedPreferences` 持久化语言偏好
- `applyLocale()` 通过 `ContextWrapper` 设置 `Configuration.locale`
- 支持 `zh`、`en`、`null`（跟随系统）三种模式

### 2.3 Application 集成

**路径**: `SkillTreeApplication.kt`

- `onCreate()` 中调用 `localeManager.applyLocale(this)`
- `onConfigurationChanged()` 中重新应用语言配置

## 三、UI 层迁移

### 3.1 Composable 文本迁移

所有 Screen 中的硬编码中文文本替换为 `stringResource(R.string.xxx)`：

| 文件 | 迁移数量 |
|------|---------|
| SkillTreeScreen.kt | 7 处 |
| NodeDetailScreen.kt | 16+ 处 |
| CreateNodeDialog.kt | 10 处 |
| SettingsScreen.kt | 全量 + 语言切换 UI |
| StatisticsScreen.kt | 11 处 |
| OnboardingScreen.kt | 2 处 |

### 3.2 语言切换 UI

**路径**: `SettingsScreen.kt` → `LanguageSelectionDialog`

- 三个选项：中文 / English / 跟随系统
- 选择后保存到 `localeManager.languageTag`
- 调用 `Activity.recreate()` 使语言变更生效

### 3.3 MainActivity 集成

- Snackbar/Dialog 事件显示从 `event.message`/`event.title` 改为 `context.getString(event.messageResId, *event.messageArgs)`
- `localeManager` 参数链传递至 SettingsScreen

## 四、错误系统重构

### 4.1 UiError 重构

**路径**: `core/ui/error/UiError.kt`

所有 `String` 字段改为 `@StringRes Int` 资源 ID：

```kotlin
data class UiError(
    @StringRes val titleResId: Int,
    @StringRes val messageResId: Int,
    val messageArgs: Array<Any> = emptyArray(),
    val action: ErrorAction? = null,
    val severity: ErrorSeverity = ErrorSeverity.ERROR
)

data class ErrorAction(@StringRes val labelResId: Int, val action: () -> Unit)

data class SnackbarEvent(
    @StringRes val messageResId: Int,
    val messageArgs: Array<Any> = emptyArray(),
    val isError: Boolean = false,
    @StringRes val actionLabelResId: Int? = null,
    val action: (() -> Unit)? = null
)

data class DialogEvent(
    @StringRes val titleResId: Int,
    @StringRes val messageResId: Int,
    val messageArgs: Array<Any> = emptyArray(),
    @StringRes val confirmLabelResId: Int = R.string.common_ok,
    val onConfirm: (() -> Unit)? = null
)
```

### 4.2 ErrorStateManager 重构

**路径**: `core/ui/error/ErrorStateManager.kt`

- `mapDomainException()` / `mapDataException()` 返回 `UiError` 使用 `R.string.error_xxx` ID
- `showSnackbar(messageResId: Int, messageArgs: Array<Any>, ...)` 
- `showErrorDialog(titleResId: Int, messageResId: Int, messageArgs: Array<Any>, ...)`

### 4.3 ViewModel 迁移

| ViewModel | 迁移内容 |
|-----------|---------|
| SkillTreeViewModel | `showSnackbar("已导出到 $path")` → `showSnackbar(R.string.tree_export_success, arrayOf(path))` |
| SettingsViewModel | 同上模式 |
| StatisticsViewModel | 同上模式 |
| NodeDetailViewModel | 同上模式 |
