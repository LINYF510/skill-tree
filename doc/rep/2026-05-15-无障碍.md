## 二、核心组件

### 2.1 AccessibilityExt.kt — 无障碍扩展函数

**路径**: `core/ui/accessibility/AccessibilityExt.kt`

提供 5 个 Modifier 扩展函数：

| 函数 | 用途 |
|------|------|
| `accessibilityLabel(label)` | 添加无障碍标签，合并后代语义 |
| `accessibilityButton(label, actionLabel?)` | 按钮语义，设置 Role.Button |
| `accessibilityExpandable(isExpanded, label)` | 展开/折叠状态描述 |
| `accessibilityToggle(isChecked, label)` | 切换语义，设置 Role.Switch |
| `accessibilityListItem(label, actionHint?)` | 列表项语义，设置 Role.Button |

所有函数使用 `semantics(mergeDescendants = true)` 合并后代语义节点。

### 2.2 AccessibilityOverlay.kt — Canvas 语义代理层

**路径**: `core/ui/accessibility/AccessibilityOverlay.kt`

核心组件——在 Canvas 上方叠加透明的 Box 组件，为每个可见节点创建无障碍代理：

- 接收 `treeNodes`、`nodePositions`、`visibleNodeIds`、`scale`、`offsetX/Y` 参数
- 为每个可见节点创建透明可点击 Box
- 手动计算屏幕坐标：`pos.x * scale + offsetX - nodeWidth * scale / 2`
- 节点描述包含：类型（能力/物质）+ 标题 + 等级 + 子节点数 + 内容状态
- 设置 `Role.Button` 和 `stateDescription`
- 支持 `onClick` 和 `onLongClick` 回调

### 2.3 SkillTreeScreen 集成

**路径**: `feature/tree/SkillTreeScreen.kt`

- 将 `nodePositions` 和 `visibleNodeIds` 计算从 Canvas 内部提升到外层 Box
- 使用 `onSizeChanged` 捕获 Canvas 尺寸
- 在 Canvas 之后添加 `AccessibilityOverlay` 组件
- 添加键盘导航支持：`focusRequester` + `focusable` + `onKeyEvent`

## 四、键盘导航

### 4.1 ViewModel 导航方法

**路径**: `feature/tree/SkillTreeViewModel.kt`

新增 4 个导航方法：

| 方法 | 功能 |
|------|------|
| `navigateToParent()` | 选中当前节点的父节点 |
| `navigateToFirstChild()` | 选中当前节点的第一个子节点 |
| `navigateToPreviousSibling()` | 选中上一个兄弟节点 |
| `navigateToNextSibling()` | 选中下一个兄弟节点 |

所有方法基于 `_selectedNodeId` 和 `_uiState.value.nodes` 的树结构计算。

### 4.2 SkillTreeScreen 键盘事件

| 按键 | 行为 |
|------|------|
| ↑ (DirectionUp) | 导航到父节点 |
| ↓ (DirectionDown) | 导航到第一个子节点 |
| ← (DirectionLeft) | 导航到上一个兄弟节点 |
| → (DirectionRight) | 导航到下一个兄弟节点 |
| Enter / Spacebar | 打开选中节点详情 |

使用 `FocusRequester` + `focusable()` + `onKeyEvent` 实现。

## 八、后续优化建议

1. **contentDescription 国际化**: 当前无障碍描述使用硬编码中文，后续应迁移到 `stringResource`
2. **AccessibilityOverlay 性能**: 大量节点时（>100），代理层可能影响性能，可考虑虚拟化
3. **TalkBack 实际测试**: 需在真机上使用 TalkBack 进行端到端验证
4. **节点选中状态**: AccessibilityOverlay 应同步 Canvas 的选中高亮状态
5. **语义树调试**: 使用 `Compose Preview` 的语义树检查工具验证语义结构
