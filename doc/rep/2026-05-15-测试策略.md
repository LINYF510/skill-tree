## 测试技术要点

- **JUnit5**: `@Nested` 分组 + `@DisplayName` 中文描述 + `@BeforeEach`/`@AfterEach` 生命周期
- **MockK**: `coEvery`/`every` 区分 suspend/普通函数，`relaxed = true` 仅在必要时使用
- **Turbine**: 用于 Flow 和 StateFlow 的异步断言
- **Truth**: `assertThat().isEqualTo()` 替代 JUnit Assert
- **协程测试**: `runTest` + `StandardTestDispatcher` + `advanceUntilIdle()`
- **ViewModel 测试**: `Dispatchers.setMain(TestDispatcher)` 替换主线程调度器

## 已知问题

1. `StandardTestDispatcher` 与 Turbine 的跨调度器交互存在时序复杂性，初始化状态流转测试改用直接值断言
2. `@OptIn(ExperimentalCoroutinesApi::class)` 需标注在 ViewModelTest 类上，因 `StandardTestDispatcher` 为实验性 API
3. `SkillTreeRepositoryImplTest` 中 `deleteNode` 和 `insertNode` 成功测试使用 `coEvery` 重新声明而非 `coVerify`，后续可优化
4. 未覆盖 UI 层 Compose 测试（需 Android 仪器测试环境）和集成测试
