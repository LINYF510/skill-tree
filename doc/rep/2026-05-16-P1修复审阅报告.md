## 三、Part B: Canvas 缩放回弹动画

### 3.1 AnimationConfig 新增常量

```kotlin
const val ZOOM_BOUNCE_DURATION = 200  // 已定义但未使用
val ZOOM_BOUNCE_SPRING = SpringSpec<Float>(
    dampingRatio = Spring.DampingRatioMediumBouncy,
    stiffness = Spring.StiffnessMedium
)
```

**评价**: `ZOOM_BOUNCE_DURATION` 未被使用（实际使用 SpringSpec 而非时长动画），属于冗余常量，建议移除。

### 3.2 手势处理改造

- ✅ `detectTransformGestures` 已替换为 `awaitEachGesture` + 手动双指距离计算
- ✅ 缩放期间不进行 coerce，允许短暂超出 MIN_SCALE/MAX_SCALE
- ✅ 手指抬起后检测是否需要回弹，通过 `Animatable.animateTo` 执行弹簧动画
- ✅ `isBounceAnimating` 标志防止回弹期间手势冲突
- ✅ 回弹使用 `SpringSpec(MediumBouncy, MediumStiffness)`，手感自然

### 3.3 潜在问题

1. ⚠️ **`ZOOM_BOUNCE_DURATION = 200` 未使用**: 冗余常量，建议移除或改为注释说明
2. ⚠️ **回弹动画在 `LaunchedEffect` 外执行**: `scaleAnimatable.animateTo` 在 `awaitEachGesture` 的代码块中执行，虽然功能正确，但需确保动画期间 Composable 不会被意外重组导致动画中断
