### 2.2 AccessibilityOverlay.kt — ✅ 通过（有注意事项）

**文件**: `core/ui/accessibility/AccessibilityOverlay.kt` (117 行)

核心实现：
- Canvas 上方叠加透明 Box 层，每个可见节点一个可点击区域
- 屏幕坐标计算：`pos.x * scale + offsetX - nodeWidth * scale / 2`
- `buildNodeDescription()`: 类型 + 标题 + 等级 + 子节点数 + 内容状态
- `buildNodeStateDescription()`: 点击/长按操作提示
- 使用 `LocalDensity` 进行 dp 转换

**问题**:
1. ⚠️ **性能隐患**: 当节点数 >100 时，每个节点创建一个 Box 组件可能导致 Compose 重组开销过大。建议后续引入虚拟化（仅渲染视口内节点）或限制最大渲染数量
2. ⚠️ **硬编码中文**: `buildNodeDescription()` 和 `buildNodeStateDescription()` 中的 "能力节点"、"物质节点"、"点击查看详情" 等为硬编码中文，未走 i18n
3. ⚠️ **onClickLabel 硬编码**: `clickable(onClickLabel = "查看 ${node.entity.title} 的详情")` 也是硬编码中文
