### Part G: 节点升级解锁动画 ✅

```kotlin
LaunchedEffect(uiState.nodes) {
    uiState.nodes.forEach { node ->
        val prevLevel = previousNodeLevels[node.id]
        if (prevLevel != null && node.level > prevLevel) {
            unlockingNodeId.value = node.id
            unlockProgress.snapTo(0f)
            unlockProgress.animateTo(1f, tween(AnimationConfig.NODE_UNLOCK_DURATION))
            // 升级时粒子数量翻倍
            if (AnimationConfig.areParticlesEnabled) {
                val system = ParticleSystem(
                    centerX = nodePos.x, centerY = nodePos.y,
                    particleCount = AnimationConfig.PARTICLE_COUNT * 2,
                    colors = listOf(Color(0xFF000000.toInt() or glowColorInt), colors.primary)
                )
                system.emit()
                particleSystems[node.id] = system
            }
            unlockingNodeId.value = null
            unlockProgress.snapTo(0f)
        }
        previousNodeLevels[node.id] = node.level
    }
}
```

- 使用 `previousNodeLevels` 追踪等级变化
- 升级时触发解锁光效 + 双倍粒子（60个）
- 粒子颜色使用等级的 glowColor + primary

### Part G: 临时文件清理 ⚠️ 需注意

提示词要求"拍照后清理临时文件"，当前实现中拍照文件保存在 `cache/camera/` 目录，通过 FileProvider 传递 URI 给 `addAttachment()`，`AttachmentFileManager.copyToInternalStorage()` 会将文件复制到内部存储。但 **cache 中的临时文件没有显式删除**。

- Android 系统会在存储空间不足时自动清理 cache 目录
- 这不是严重问题，但与提示词要求有轻微偏差

**严重度**: P3（低），系统自动清理机制可覆盖
